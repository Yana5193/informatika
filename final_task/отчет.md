# Отчет по проекту «MentalStat»
## Система мониторинга и анализа психологического состояния персонала 

### 1. Целевая аудитория и задачи 
Проект ориентирован на три ключевые группы пользователей:

* **Сотрудники (Респонденты):** Источник первичной информации.  Используют систему для регулярной самодиагностики, что обеспечивает конфиденциальность и минимизирует стресс.
* **Штатный психолог (Главный эксперт):** Основной потребитель аналитики. Использует систему для автоматизированного скрининга и выявления сотрудников в «красной зоне» (критическое состояние). 
* **HR-менеджеры и руководство:** Используют агрегированные отчеты для анализа состояния подразделений и принятия управленческих решений. 

### 2. Анализ конкурентных решений 

| Критерий | Happy Job | Yva.ai | MentalStat (Проект) |
| :--- | :--- | :--- | :--- |
| **Метод оценки** | Активный (опросы) | Пассивный (ИИ-анализ) | Активный (научные методики) |
| **Целевой пользователь** | HR-директор | CEO / Собственник | Штатный психолог |
| **Безопасность** | Облачное хранилище | Облачное хранилище | Локальная БД (SQLite) |
| **Детализация** | Групповые отчеты | Прогноз увольнений | Индивидуальный профиль |

---

* **Happy Job:** Лидер российского рынка, отлично измеряет лояльность (eNPS), но предоставляет усреднённые данные.Программа не ориентирована на клиническую работу с выгоранием.
* **Yva.ai:** Использует методы пассивного анализа метаданных почты и мессенджеров. Вызывает этические вопросы и требует дорогой настройки нейросетей. 
* **BambooHR:** Глобальная HRM-система.Главный минус — хранение данных на зарубежных серверах, что критично для психологической информации. 

### 3. Описание реализуемого процесса 
Процесс автоматизации включает следующие этапы: 
1. **Авторизация:** Разграничение прав доступа (Сотрудник / Психолог / Менеджер). 
2. **Тестирование:** Выбор методики и последовательный ответ на вопросы. 
3.  **Обработка:** Автоматический подсчет баллов и сохранение в БД SQLite. 
4.  **Аналитика:** Сопоставление баллов с таблицей рекомендаций и визуализация данных. 

---

### 4. Схема данных 

CREATE TABLE Departments (id_dept INTEGER PRIMARY KEY, title TEXT);
CREATE TABLE Employees (id_emp INTEGER PRIMARY KEY, full_name TEXT, dept_id INTEGER, password TEXT);
CREATE TABLE Test_List (id_test INTEGER PRIMARY KEY, test_name TEXT);
CREATE TABLE Questions (id_quest INTEGER PRIMARY KEY, test_id INTEGER, quest_text TEXT);
CREATE TABLE Test_logs (id_log INTEGER PRIMARY KEY, emp_id INTEGER, test_id INTEGER, date_passed DATETIME);
CREATE TABLE Mental_Scores (id_score INTEGER PRIMARY KEY, log_id INTEGER, total_points INTEGER, status TEXT);

5. **Аналитика и отчеты**
Общая таблица результатов (Психолог/Менеджер) 

SELECT e.full_name, d.title, e.position, t.test_name, ms.total_points, tl.date_passed 
FROM Employees e 
JOIN Departments d ON e.dept_id = d.id_dept 
JOIN Test_logs tl ON e.id_emp = tl.emp_id 
JOIN Test_List t ON tl.test_id = t.id_test 
JOIN Mental_Scores ms ON tl.id_log = ms.log_id;


Средний балл по отделам для графика 

SELECT d.title, AVG(ms.total_points) as avg_score 
FROM Departments d JOIN Employees e ON d.id_dept = e.dept_id 
JOIN Test_logs tl ON e.id_emp = tl.emp_id 
JOIN Mental_Scores ms ON tl.id_log = ms.log_id 
GROUP BY d.title ORDER BY avg_score DESC;
